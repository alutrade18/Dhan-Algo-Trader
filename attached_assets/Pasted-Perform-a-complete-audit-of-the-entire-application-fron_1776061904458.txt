Perform a complete audit of the entire application (frontend + backend).

Your tasks:

1. Logs & Backend Validation
- Check every page, module, and component in the app.
- Ensure each page is properly connected to backend logging.
- Verify that all API calls are logged with:
  - request payload
  - response data
  - status codes
  - error messages (real-time)
- Identify any page or feature where logging is missing or incomplete.

2. API Endpoint Verification
- Validate that every API call is using the correct and updated endpoint.
- Detect outdated, broken, or unused API integrations.
- Ensure proper error handling is implemented for each API:
  - HTTP errors (4xx, 5xx)
  - network failures
  - timeout handling

3. Logs Page Accuracy
- Cross-check the Logs page with actual backend activity.
- Ensure logs shown in UI match real backend events.
- Verify real-time updates (no stale or cached logs).
- Confirm proper formatting and categorization of logs (info, warning, error).

4. No Hardcoding Policy
- Scan the entire codebase and identify any hardcoded:
  - API URLs
  - credentials
  - tokens
  - business logic values
- Replace them with dynamic/config-based or database-driven values.

5. Centralized API Configuration
- Ensure all API-related configurations are not scattered.
- Move all API configs into a centralized system.

6. Separate Database for API Config & Logs
- Design and enforce a separate database/schema for:
  - API configurations (endpoints, headers, auth, rate limits)
  - API logs (requests, responses, timestamps, errors)
- Ensure backend reads API details dynamically from DB (no hardcoding).

7. Missing Code Detection
- Identify missing backend integrations, incomplete functions, or broken flows.
- Highlight any UI that is not properly wired to backend.

8. Output Required
Provide a structured report with:
- Missing logging areas
- Broken or outdated APIs
- Hardcoded elements found
- Suggested fixes with code examples
- Database schema suggestion for API config + logs
- Final status: (Safe / Needs Fix / Critical)

Important Rules:
- Do NOT hardcode anything.
- Follow scalable SaaS architecture (multi-user ready).
- Keep everything modular and production-ready.
