BUG #1 — market-feed-ws.ts: Binary packet byte offset collision
FILE: artifacts/api-server/src/lib/market-feed-ws.ts
PROBLEM: In parsePacket(), two variables read from the SAME byte offset:
  const exchId = buf.readUInt8(4);       // reads byte at offset 4
  const securityId = buf.readInt32LE(4); // ALSO reads 4 bytes starting at offset 4
This means exchId is always the first byte of the securityId integer — completely wrong.
According to Dhan's binary feed protocol: responseCode is at offset 0 (1 byte),
exchange segment ID is at offset 1 (1 byte), and security_id is at offset 4 (4 bytes LE).
FIX: Change parsePacket() to:
  const responseCode = buf.readUInt8(0);
  const exchId = buf.readUInt8(1);          // ← FIX: offset 1, not 4
  const securityId = buf.readInt32LE(4);    // unchanged
  const exchSeg = EXCHANGE_MAP[exchId] ?? "UNKNOWN";

BUG #6 — order-guards.ts: maxDailyLoss and maxDailyProfit limits are NEVER enforced
FILE: artifacts/api-server/src/lib/order-guards.ts
PROBLEM: Settings has maxDailyLoss and maxDailyProfit fields, and the UI shows them in
Risk Manager. But runOrderGuards() never checks these limits. Orders keep getting placed
even when the daily loss limit is breached.
FIX: Add a new guard function checkDailyPnlLimits() and call it in runOrderGuards():

  export async function checkDailyPnlLimits(): Promise<OrderGuardResult> {
    const settings = await getSettings();
    const maxLoss = settings?.maxDailyLoss ? Number(settings.maxDailyLoss) : null;
    const maxProfit = settings?.maxDailyProfit ? Number(settings.maxDailyProfit) : null;
    if (!maxLoss && !maxProfit) return { allowed: true };

    try {
      const positions = await dhanClient.getPositions() as Array<Record<string, unknown>>;
      const todayPnl = Array.isArray(positions)
        ? positions.reduce((s, p) => s + Number(p.realizedProfit || 0) + Number(p.unrealizedProfit || 0), 0)
        : 0;

      if (maxLoss && todayPnl <= -maxLoss) {
        return { allowed: false, reason: `Daily loss limit of ₹${maxLoss.toLocaleString("en-IN")} reached. Today's P&L: ₹${todayPnl.toFixed(2)}` };
      }
      if (maxProfit && todayPnl >= maxProfit) {
        return { allowed: false, reason: `Daily profit target of ₹${maxProfit.toLocaleString("en-IN")} reached. Today's P&L: ₹${todayPnl.toFixed(2)}` };
      }
    } catch {
      return { allowed: true };
    }
    return { allowed: true };
  }

Then call it inside runOrderGuards() before the position size check.
