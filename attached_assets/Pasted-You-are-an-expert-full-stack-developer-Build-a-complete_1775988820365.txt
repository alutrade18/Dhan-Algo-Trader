### 3A: Live Market Feed WebSocket
File: artifacts/api-server/src/lib/market-feed-ws.ts

Connect to: wss://api-feed.dhan.co?version=2&token=JWT&clientId=ID&authType=2

Supports 3 subscription modes:
- RequestCode 15 = Ticker (LTP only)
- RequestCode 17 = Quote (LTP + OHLCV)
- RequestCode 21 = Full (LTP + OHLCV + 5-level depth)

Binary packet parsing:
- Response Code 2 (Ticker): bytes 8-11 = float32 LTP, bytes 12-15 = int32 LTT
- Response Code 4 (Quote): LTP + volume + OHLCV (complete parse)
- Response Code 8 (Full): LTP + OHLCV + 5 bid/ask depth levels (each 20 bytes)
- Response Code 6 (Prev Close): previous day close
- Response Code 50 (Disconnect): with disconnect reason code

Max 5 connections, 5000 instruments per connection.
Max 100 instruments per subscribe message.
Auto-reconnect after 5 seconds on disconnect.
Ping-pong: server sends ping every 10s, auto pong.

Emit events: "tick", "quote", "depth", "prevClose", "connected", "disconnected"

### 3B: Live Order Update WebSocket
File: artifacts/api-server/src/lib/order-update-ws.ts

Connect to: wss://api-order-update.dhan.co
After connect, send auth message:
{
  "LoginReq": { "MsgCode": 42, "ClientId": "clientId", "Token": "JWT" },
  "UserType": "SELF"
}

Messages are JSON (not binary).
On order update received → emit "orderUpdate" event with full order data.
Fields: OrderNo, Status, TxnType (B/S), Product, Symbol, Quantity, TradedQty, TradedPrice, etc.

### 3D: Frontend Socket.io Client
File: artifacts/trading-platform/src/lib/market-socket.ts

```typescript
import { io } from "socket.io-client";

class MarketSocket {
  private socket = io(window.location.origin);
  private tickListeners = new Map<string, Set<Function>>();

  subscribe(exchange: string, securityId: string, cb: Function, mode = "ticker") {
    const key = `${exchange}:${securityId}`;
    if (!this.tickListeners.has(key)) {
      this.tickListeners.set(key, new Set());
      this.socket.emit("market:subscribe", { exchange, securityIds: [securityId], mode });
    }
    this.tickListeners.get(key)!.add(cb);
  }

  unsubscribe(exchange: string, securityId: string, cb: Function) { ... }
  
  onOrderUpdate(cb: Function) {
    this.socket.on("order:update", cb);
    return () => this.socket.off("order:update", cb);
  }
}

export const marketSocket = new MarketSocket();
```

### Market hours check:
```typescript
function isNSEMarketHours(): boolean {
  const now = new Date(new Date().toLocaleString("en-US", { timeZone: "Asia/Kolkata" }));
  const day = now.getDay();
  const h = now.getHours(), m = now.getMinutes();
  const minutes = h * 60 + m;
  return day >= 1 && day <= 5 && minutes >= 9 * 60 + 15 && minutes <= 15 * 60 + 30;
}
```

### Strategy engine flow:
1. Fetch all active strategies from DB
2. For each: check market hours, check kill switch, check daily loss/profit limits

4. Calculate indicators, evaluate conditions
5. If entry conditions met → place INTRADAY order via dhanClient.placeOrder()
6. Save to trade_logs, update strategy stats, auto-pause strategy
7. Send Telegram alert

### 7F: Positions Page (/positions)
INTRADAY positions only. Filter positionType != CLOSED.
Live LTP via WebSocket subscription.

Columns:
- Symbol | Exchange | Side (LONG/SHORT) | Qty | Avg Buy | Avg Sell | LTP | Unrealized P&L | Realized P&L | Actions

Live unrealized P&L calculation:
- LONG: (LTP - buyAvg) * netQty
- SHORT: (sellAvg - LTP) * netQty

Actions per position:
- "Exit" button → place opposite INTRADAY MARKET order for netQty
- "Add" button → place same direction INTRADAY order

Bottom bar:
- Total Unrealized P&L (live)
- Total Realized P&L
- "EXIT ALL" red button (calls DELETE /api/positions with confirmation)

Auto-refresh every 30s + WebSocket LTP updates.
