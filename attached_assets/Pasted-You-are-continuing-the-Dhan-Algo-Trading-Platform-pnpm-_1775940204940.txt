### Tab 1: Today's Orders

Fetch from: GET /api/orders (Dhan API, already proxied in backend)

Show a table with these columns:
| Time | Symbol | Type | Product | Qty | Price | Trigger | Status | Action |

- Time: show createTime formatted as "HH:MM:SS"
- Symbol: show tradingSymbol (bold, monospace font)
- Type: BUY badge (green) or SELL badge (red)
- Product: INTRADAY / CNC / MARGIN / MTF badge (gray pill)
- Qty: show quantity, if filledQty > 0 show "10/5 filled" format
- Price: show price, if MARKET order show "MKT" text
- Trigger: show triggerPrice only if > 0, else show "-"
- Status: colored badge:
  - TRADED → green
  - PENDING / TRANSIT → amber/yellow
  - REJECTED → red
  - CANCELLED → gray
  - EXPIRED → gray
  - PART_TRADED → blue
- Action column: show buttons based on status:
  - If PENDING or TRANSIT: show "Modify" button (blue, small) + "Cancel" button (red, small)
  - If TRADED/CANCELLED/REJECTED/EXPIRED: show "-"

Auto-refresh every 15 seconds (show a small countdown timer "Refreshing in 12s" in top right of table).

Show loading skeleton while fetching.
Show empty state if no orders: "No orders placed today" with an icon.

### Modify Order Modal (Dialog)

When user clicks "Modify" button, open a Dialog with:
- Order ID shown at top (read-only)
- Symbol shown (read-only)
- Current values pre-filled

Editable fields:
- Order Type: Select dropdown → LIMIT / MARKET / STOP_LOSS / STOP_LOSS_MARKET
- Quantity: number input (required)
- Price: number input (required if LIMIT or STOP_LOSS, disabled if MARKET)
- Trigger Price: number input (required only if STOP_LOSS or STOP_LOSS_MARKET)

Submit button: "Modify Order" → calls PUT /api/orders/:orderId
On success: show toast "Order modified successfully", close modal, refresh table
On error: show error toast with omsErrorDescription if available
